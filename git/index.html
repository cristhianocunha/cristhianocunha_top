Configuração do Git e GitHub no Windows 10/11

Nota: Este guia contém botões para copiar comandos diretamente. Clique no botão "Copiar" ao lado de cada bloco de código.

1. Instalação do Git

1.1 Baixe o Git

Acesse o site oficial: https://git-scm.com/download/win

O download começará automaticamente para a versão adequada ao seu Windows (32-bit ou 64-bit)

1.2 Execute o instalador

Dê duplo clique no arquivo baixado (ex: Git-2.xx.x-64-bit.exe)

Siga as instruções do instalador

1.3 Configuração durante a instalação

  • Aceite a licença
  • Selecione o local de instalação (pode deixar o padrão)
  • Na seleção de componentes, marque:
    • "Git Bash Here"
    • "Git GUI Here"
    • "Associate .git* configuration files with the default text editor"
  • Escolha o editor padrão (recomendo "Use Visual Studio Code as Git's default editor" se tiver o VSCode instalado)
  • Na configuração do PATH, selecione: "Git from the command line and also from 3rd-party software"
  • Para conversão de linha final (line endings), selecione: "Checkout Windows-style, commit Unix-style line endings"
  • Configure o terminal: selecione "Use MinTTY (the default terminal of MSYS2)"
  • Nas opções extras, marque:
    • "Enable file system caching"
    • "Enable Git Credential Manager"

1.4 Conclua a instalação

Clique em "Install" e aguarde a conclusão da instalação.

2. Configuração inicial do Git

2.1 Abra o Git Bash

Menu Iniciar > Git > Git Bash

2.2 Configure seu usuário

Substitua com seus dados:

git config --global user.name "Seu Nome"
git config --global user.email "seu@email.com"

2.3 Configure o editor padrão (opcional)

Se quiser usar o VSCode como editor padrão:

git config --global core.editor "code --wait"

2.4 Verifique as configurações

git config --list

3. Configuração da conexão com o GitHub

Opção 1: Usando HTTPS (mais simples)

3.1.1 Gere um token de acesso pessoal

  • Acesse https://github.com/settings/tokens
  • Clique em "Generate new token"
  • Dê um nome descritivo
  • Selecione os escopos necessários (para repositórios privados, marque "repo")
  • Clique em "Generate token"
  • Copie o token gerado (ele só será mostrado uma vez)
Atenção: Guarde este token em um local seguro, pois você não poderá vê-lo novamente depois de sair da página.

Opção 2: Usando SSH (recomendado para maior segurança)

3.2.1 Gere uma chave SSH

ssh-keygen -t ed25519 -C "seu@email.com"

Pressione Enter para aceitar o local padrão

Digite uma senha segura (opcional, mas recomendado)

3.2.2 Adicione a chave ao ssh-agent

eval $(ssh-agent -s)
ssh-add ~/.ssh/id_ed25519

3.2.3 Adicione a chave pública ao GitHub

  • Mostre a chave pública:
    • cat ~/.ssh/id_ed25519.pub
  • Copie todo o conteúdo
  • Acesse https://github.com/settings/keys
  • Clique em "New SSH key"
  • Cole a chave e salve

3.2.4 Teste a conexão

ssh -T git@github.com

Responda "yes" se perguntado sobre continuar a conexão

Você deve ver uma mensagem de sucesso

4. Configurando o Git Credential Manager (opcional)

O Git Credential Manager já foi instalado automaticamente com o Git. Ele irá:

  • Armazenar suas credenciais de forma segura
  • Gerenciar automaticamente a autenticação com o GitHub

Para verificar se está funcionando:

git config --global credential.helper

Deve retornar "manager" ou algo similar.

5. Testando sua configuração

5.1 Crie um repositório de teste

mkdir teste-git
cd teste-git
git init
echo "# Teste Git" > README.md
git add README.md
git commit -m "Primeiro commit"

5.2 Crie um repositório no GitHub

  • Acesse https://github.com/new
  • Dê um nome (ex: "teste-git")
  • Não marque "Initialize this repository with a README"

5.3 Conecte seu repositório local ao GitHub

Para SSH:

git remote add origin git@github.com:seu-usuario/teste-git.git

Para HTTPS:

git remote add origin https://github.com/seu-usuario/teste-git.git

5.4 Envie suas alterações

git push -u origin main

Se for solicitado, insira seu nome de usuário do GitHub e:

  • Para HTTPS: use o token de acesso como senha
  • Para SSH: use a senha da sua chave SSH (se você configurou uma)

Pronto!

Seu Git e GitHub estão configurados corretamente no seu Windows.

Para salvar como PDF, use a função de impressão do seu navegador (Ctrl+P) e selecione "Salvar como PDF".
